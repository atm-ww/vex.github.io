  1. 需求澄清： 我們會先詳細討論您的想法與目標，確認需求的範圍與可行性。
  2. 估時與報價： 根據需求的複雜度，我會提供詳細的開發時程估算與報價。

  4. 內部測試： 在交付前，我會進行完整的內部測試，確保功能的穩定性。
  5. 交付驗收： 交付資源給您進行測試與驗收，並根據回饋進行調整。
  6. 文件與迭代： 交付最終版本時，會附上必要的文件。後續可討論長期維護或新功能的迭代開發。

作品案例

案例一：伺服器延遲優化

問題： 伺服器在線人數超過 80 人時，全體玩家 ping 值飆高，事件處理延遲嚴重。

解法： 逐一分析高資源佔用腳本，重構了幾個頻繁觸發的 server-side 事件，並將部分邏輯改為 client-side 處理。

結果： 伺服器在 128 人滿載情況下，resmon 平均值從 0.8ms 降至 0.3ms，事件延遲感明顯消失。

性能 QBCore

案例二：客製化車輛 HUD

問題： 客戶需要一個與其伺服器風格一致，且功能簡潔的車輛儀表板 UI。

解法： 使用 ox_lib 作為基礎，設計並開發了一個新的 NUI 介面，僅顯示速度、油量和引擎狀態，並加入微動畫效果。

結果： 交付了一個低資源佔用 (0.01ms) 且視覺風格統一的 HUD，並提供配置文件讓客戶自行調整顏色。

UI ox_lib

案例三：Discord 玩家數據同步

問題： 管理員希望在 Discord 中能快速查詢玩家的遊戲內資產。

解法： 開發了一個 Lua 腳本，監聽玩家數據變化，並透過 Webhook 將加密後的數據發送到一個自製的 Discord Bot。

結果： 管理員可透過簡單的 Discord 指令，即時查詢玩家的銀行餘額、車輛等資訊，大幅提升管理效率。

常見問題

開發一個功能大概需要多久？

時程取決於功能的複雜度。一個簡單的 UI 介面可能需要 3-5 個工作天，而一個複雜的系統（如自訂義職業）可能需要 2-3 週。我會在需求澄清階段提供詳細的時程估算。

我可以要求修改幾次？

在開發過程中，每個里程碑交付後，會提供 1-2 次的微調機會。若有重大變更，則需重新評估時程與費用。

交付後有保固嗎？

交付後提供為期 14 天的保固期，期間內若因我方程式碼造成的 Bug，將免費修復。但不包含因伺服器環境變更或與其他資源衝突造成的問題。

會提供原始碼嗎？

是的，所有客製化開發的專案，在結案後都會交付完整的、無加密的原始碼。
